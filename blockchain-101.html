01 / The building block

Hashing: the fingerprint machine

A hash function takes any input — a word, a book, a transaction — and returns a fixed-length fingerprint. Change one character and the entire fingerprint changes. It's easy to compute forward, and effectively impossible to reverse.

—

Try adding a single letter, or changing "b" to "B". Watch how much of the fingerprint flips.

#

Why this matters

Hashes are deterministic (same input → same output), fast to check but slow to reverse, and avalanche-sensitive (a tiny change scrambles everything). These three properties are what make a blockchain tamper-evident.

02 / What's inside

Anatomy of a block

A block is a container. It bundles a batch of transactions with a little metadata, and — crucially — the hash of the block before it. That backward pointer is what turns a pile of blocks into a chain.

↩

Previous hash

The fingerprint of the block that came before. This is the "chain." Break it and everything after collapses.

≡

Transactions

The actual payload — who sent what to whom. Often summarized into a single "Merkle root" fingerprint.

◷

Timestamp & nonce

When the block was made, plus the lucky number miners search for to satisfy the difficulty target.

#

Block hash

A fingerprint of everything above. It's the block's identity and the next block's "previous hash."

03 / The signature idea

The chain — and why you can't cheat it

Here is a real, working blockchain. Each block stores the previous block's hash. Edit any block's data and watch it — and every block after it — turn red instantly. To fix the tamper you'd have to re-mine every block that follows, all faster than the rest of the network. That's the whole security model.

Difficulty is set to 3 leading zeros here so mining stays fast. Real Bitcoin requires ~19 leading zeros — trillions of guesses per block.

04 / Agreeing without a boss

Consensus: who gets to add the next block?

With no central authority, the network needs a fair, expensive-to-fake way to pick who writes the next page. The two dominant answers are Proof of Work and Proof of Stake.

used by · Bitcoin

Proof of Work

Miners race to guess a nonce that produces a valid hash. First to solve it wins the block reward. Cheating means out-computing the entire planet.

Validators lock up ("stake") coins as collateral. The protocol picks one at random to propose the block. Cheat, and your stake gets slashed.

05 / No single point of failure

Decentralization & the network

There's no central server. Instead, thousands of independent computers ("nodes") each keep a full copy of the ledger. When you make a transaction, it ripples across the network until everyone agrees. Press the button and watch a transaction propagate.

◎

Every node is a referee

Each node independently checks every block against the same rules. Try to sneak in an invalid transaction and honest nodes simply reject it.

51

The 51% problem

To rewrite history you'd need to control the majority of the network's mining power (PoW) or staked value (PoS) — astronomically expensive, and you'd wreck the asset you're attacking.

∞

Permissionless

Anyone can run a node, send a transaction, or read the entire history. No sign-up, no gatekeeper.

06 / Owning things

Keys, wallets & digital signatures

You don't "log in" to a blockchain. You prove ownership with math. A private key is a secret number; its matching public key is your address. You sign transactions with the private key, and anyone can verify the signature with the public one — without ever seeing your secret. Generate a real keypair below and try to forge a signature.

08 / Built on top

Smart contracts & what people build

Once you can run code on a blockchain, a whole world opens up. These are the categories you'll hear about most — each is just a smart contract (or many) doing a specific job.

λ

Smart contracts

Self-executing programs. "If X happens, do Y" — enforced by the network, no middleman needed. The foundation for everything below.

◈

Tokens & NFTs

Contracts that track ownership. Fungible tokens (like stablecoins) are interchangeable; NFTs are unique, one-of-a-kind items.

⇄

DeFi

Decentralized Finance — lending, trading and earning interest through code instead of banks. Runs 24/7, open to anyone.

▤

DAOs

Decentralized Autonomous Organizations — internet-native groups that vote and manage shared funds via smart contracts.

⚡

Layer 2s

Networks like Arbitrum, Optimism & Base that bundle thousands of transactions and post a summary to Ethereum — cheaper and faster.

⛽

Gas fees

Every action costs "gas," paid in ETH, to compensate validators for computation. Busy network = higher gas.
